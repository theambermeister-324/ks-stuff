Objection Handling

Anticipated objections and responses based on demo call

"I don't see the benefit for complex components"

Button was intentionally simple. Complex components with slots, state, and props are where Knapsack shines — live rendering, composability, spec alignment, copy-paste production code.

Reframe: Value scales with complexity. Simple proves concept; complex proves ROI.
"Where does source code live?"

Knapsack reads from your existing repo via npm or local files. Your code doesn't move. It's a rendering layer, not a repository.

Reframe: Code stays where it is. Knapsack presents it in context with designs and docs.
"Budget is tight / we don't have developers yet"

Non-technical wiring through browser UI. FDE (Bloom) bridges gaps. Angular devs available sooner. Set up the workflow before the team arrives — cheaper than retrofitting.

Reframe: Build the highway before the cars arrive. FDE support is included.
"Compliance concerns about repo structure"

Both approaches (same/separate repos) used by financial services customers. Configuration choice, not a compliance constraint.

Reframe: Solved at other FinServ firms. Happy to share patterns.

Phase 1 — Design-to-Code & Faster Workflows
Design-to-Code Workflow + Component Wiring
Next

Addresses: Fragmented design-to-dev handoff + No documented developer workflow

DS Developer Workflow (Jeff's Team):
1. Designer creates component in Figma
2. Knapsack shows Figma spec alongside empty code slot
3. Developer builds React/Angular component
4. Wires component to Knapsack via npm or local path
5. Spec alignment check: Figma props vs coded props
6. PR-based review, merge to production
7. Component published with live rendering + docs
Product Developer Workflow (Consumers):
1. Product developer receives design from Figma
2. Opens Knapsack to find components used in design
3. Browses demos/templates for correct attribute sets
4. Copies production-ready code from Knapsack
5. Composes complex layouts using slots
6. References usage docs and accessibility guidelines
7. Ships with confidence — code matches design intent
Component Wiring Sprint: Connect Ed Jones' existing 30+ components to Knapsack. Bloom (FDE) can lead initial wiring through the browser UI — non-technical process. Start with Angular (available resources), then React.
Figma Integration: Connect Ed Jones Figma component library. Enable spec alignment view so designers see coded status alongside design. Support Wendi's accessibility documentation flow.
Ed Jones Outcome: New developers onboard into a governed system with clear patterns, live code examples, and copy-paste production code. Sal's design side and Jeff's engineering side have a shared source of truth. Documentation fragmentation reduced.
Owner: Bloom (FDE) + Jeff's team | Timeline: 4-6 weeks | Aligns to: Jeff's team onboarding
Phase 2 — Governance & Guardrails
Governance, Accessibility & Guardrails
Following

Addresses: A11y doc fragmentation + "Collect and curate" contribution governance

Accessibility Documentation Workflow:
1. Wendi authors accessibility docs in Knapsack (semantic structure, screen reader support)
2. Guidelines attached to components at the system level
3. Figma plugin surfaces contextual accessibility info
4. Developers see accessibility requirements alongside code
5. Annual audit process (~30 components) tracked in system
6. WCAG 2.2 compliance roadmap visible to stakeholders
Contribution & Curation Workflow:
1. Product team builds component to design standards
2. DS team reviews and identifies "best in show" work
3. Component extracted and abstracted for system inclusion
4. Quality standards applied (design, reusability, a11y, tech)
5. Documentation authored in Knapsack
6. PR-based review and merge
7. Published to system with governed access
Permissions & Controls: Configure brand enforcement rules, accessibility requirements, and pattern compliance. Component Review Board uses Knapsack as decision platform. Escalation path: Review Board → Council → GP Leader.
Offshore Team Enablement: Centralized design system accessible to growing offshore teams. Clear documentation, usage guidelines, and component inventory reduce onboarding time and clarification requests.
Ed Jones Outcome: Accessibility documentation flows from Knapsack to Figma. "Collect and curate" contribution model is operationalized. Offshore teams onboard 50% faster. Wendi's gatekeeper role is supported by tooling, not just meetings.
Owner: Amber (CX) + Wendi + Sal | Timeline: 4-8 weeks | Aligns to: WCAG 2.2 year-end target

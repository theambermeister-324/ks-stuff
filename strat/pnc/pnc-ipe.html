Program Vision

Mission: Accelerate PNC's design system maturity from Level 3 (Fragmented Implementation) to Level 4 (Scaled Product System) with MCP/AI readiness‚Äîwhile ensuring Product Teams continue delivering BAU commitments without disruption.

Core Diagnosis: PNC does not have a "design system problem." PNC has a system-of-systems activation problem. The platform exists, but the control plane isn't operating. This engagement builds the governed engine that makes intelligence safe.

Engagement Model: A Design Maturity Assessment establishes baseline placement, followed by four maturity phases spanning 24 months‚Äîstabilization and governance in Year 1, execution enablement and intelligence activation in Year 2. Modernization comes before intelligence. Trust gates everything.

Four Maturity Phases

Phase 1: Stabilize & Complete Implementation

Level 3 ‚Üí Operational | Make the platform reliably usable without heroics. Resolve workspace stabilization, entitlements, repo access, publishing blockers. Teams can publish + update without Knapsack intervention.

Exit Criteria: Mobile or Ledger team can publish independently ¬∑ Workspace 2 live or date-certain ¬∑ Access no longer blocking work

Phase 2: Control Plane & Governance Activation

Level 3 ‚Üí Level 4 | Turn "systems" into a system of record with decision authority. Resolve fragmented governance, token misalignment, cross-team disconnects, "who decides what?" ambiguity.

Exit Criteria: Token strategy approved and enforced ¬∑ Governance model active beyond Ledger ¬∑ Clear rules for Mobile / Ledger / PB interaction

Phase 3: Execution Enablement & Scaled Delivery

Level 4 Operational | Prove that governed intent can safely become production change. Resolve CI/CD trust gaps, component drift, upgrade risk, "looks good but breaks downstream" issues.

Exit Criteria: One meaningful change ships end-to-end via Knapsack ¬∑ Upgrade path validated ¬∑ Teams trust system to enforce standards

Phase 4: Signals, Intelligence & MCP Pilot

Level 4 ‚Üí Early Level 5 | Introduce intelligence only after trust exists. Resolve lack of visibility into value, enable safe AI experimentation with explicit guardrails.

Exit Criteria: Signals prove faster, safer change ¬∑ MCP adds value without risk ¬∑ Execs can articulate ROI + next bets

MCP is explicitly out of scope until Phase 3 exit criteria are met. When activated in Phase 4: limited to 1-2 use cases, read-only or advisory first, no uncontrolled generation, must operate against governed context only.

Operating Rules

These rules govern how the engagement operates. They ensure work compounds instead of thrashing.

1. Maturity Gates All Work

Every epic/story must map to a maturity phase: Implementation, Control Plane, Execution, or Intelligence. If it doesn't advance maturity, it is de-prioritized.

2. PI Planning Reflects Maturity

Leadership must agree: modernization + governance come before intelligence. Volume of work is secondary to maturity advancement.

3. Exit Criteria Are Non-Negotiable

Phases advance only when exit criteria are met. No skipping ahead. No "we'll fix governance later." Trust compounds or it doesn't.

4. MCP Is Earned, Not Granted

MCP activation requires Phase 3 exit criteria met. Read-only first. 1-2 use cases maximum. Must operate against governed context only.

Key Insights & Objectives

Workshop Learnings and Strategic Alignment

What We Learned

Teams need structured pre-work, early alignment on vision, and clarity on system-wide direction before detailed planning can begin. The program assumes all product teams continue to ship their roadmaps, while Knapsack and an FDE-led architecture track take on system-level modernization, token strategy, and governance.

Key Insights

The pilot surfaced important learnings. We still default to reactive planning—timeboxing drove "what's on fire now" rather than future-looking dependency planning.

Pre-work Essential

Teams need dedicated time before workshops to review boards, align on BAU vs. modernization scope, and come prepared.

Consolidation Clarity

Private Bank struggles to plan migration without a clearer path for Ledger + PB consolidation.

Separate Planning from Solutioning

Complex topics (table deconstruction, a11y debt) need separate deep dives so PI Planning focuses on commitments.

Business Value Linkage

Clearer connection needed between objectives and leadership metrics: risk reduction, time-to-market, efficiency.

Board Clarity Needed

Boards should clearly distinguish committed work, in-flight dependencies, and exploratory ideas.

Format Inclusivity

Content and facilitation must better support designers, developers, in-room and remote participants equally.

Six strategic objectives align cross-functional teams toward shared outcomes.

Token Strategy & Density

Tokens require dedicated collaboration time—separate focused working sessions are needed to define strategy, pilots, and success measures. Phased Rollout: Color (Q1) → Spacing (Q2) → Typography (Q3)

Jira & Dependency Matrix Integration

Traceability from Objectives to Execution

Traceability Framework

Together, the updated Jira structure and Dependency Matrix give PI Planning teams a clearer, traceable view from objectives → features → teams → dependencies, and set a stronger foundation for the next PI cycle. Actions are captured as Jira "Program Board Action Items" so risks, decisions, and follow-ups are visible across teams.

Jira CSV Updates STRUCTURE

The updated Jira CSV tags every Epic and Story to one of the six PI Objectives, creating clear alignment between work items and strategic goals.

  • Objective Tagging: Every Epic/Story mapped to Modernization, Enhancement, Accessibility, Token Strategy, Migration/Docs, or Maturity & AI Readiness
  • Scope Distinction: Clear separation between new-scope work and existing implementation work
  • Commitment Levels: High-confidence commits vs. items needing collaboration vs. exploratory work
  • Program Board Actions: Action items traced with owners and target dates

Dependency Matrix CRITICAL PATHS

The Dependency Matrix highlights critical cross-team dependencies that must be actively managed to avoid delivery risk.

  • Maturity Score → Roadmap Depth: Assessment results inform prioritization and investment sequencing
  • FDE Assignment → System Progress: Architecture track capacity gates token and consolidation work
  • Angular/M3 → Components: Table and component work blocked until M3 foundations complete
  • Token Strategy → Theming: Density and branding decisions cascade to all product teams

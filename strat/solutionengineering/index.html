SE Rapid Technical Qualification – 1 Pager

Use this checklist to quickly determine if a customer is technically ready for Knapsack and to spot early blockers that will slow or kill an enterprise deal.

1. Code Framework Compatibility

✓ Go
  • React
  • Vue
  • Web Components
  • Angular
  • Twig
  • Handlebars
⚠ Maybe
  • AEM (React/Web Components)
  • PHP with Twing
  • Astro
  • Kotlin/SwiftUI (future)
  • Custom template engines
✕ No-Go
  • Xamarin
  • AEM HTL only
  • Unsupported native stacks
  • Legacy Flash
  • Proprietary frameworks

2. Build / Compilation Toolchain

✓ Go
  • Webpack
  • Vite
  • ESBuild
  • Rollup
  • Parcel
⚠ Maybe
  • Custom build scripts
  • Gradle/Maven variants
  • Make-based builds
  • Limited output capability
✕ No-Go
  • Closed proprietary toolchain
  • No build artifact output
  • Monolithic build system
  • Vendor lock-in patterns

3. Design System Maturity

Ready for Knapsack

  • Documented component guidelines
  • Component library in production
  • Token system in place
  • Clear governance model
  • Dedicated design system team

Timeline: 3-6 months typical implementation

Requires some setup

  • Some components exist but scattered
  • Process is unclear or ad-hoc
  • Documentation is inconsistent
  • No formal governance
  • Shared responsibility across teams

Timeline: 6-9 months (add token setup phase)

Heavy lift required

  • No formal design system
  • Inconsistent patterns across properties
  • Fragmented component implementations
  • No centralized token management
  • High technical debt

Timeline: 12+ months (foundation building first)

4. Quick Qualification Questions

Question Response
Can you auto-generate component code from design changes? Yes = ReadyNo = Blocker
Do design tokens exist in code? Yes = ReadyPartial = Phase 1
Single primary tech stack? Yes = ReadyMultiple = Plan
Dedicated design system team or owner? Yes = ReadyNo = Establish first

5. Decision Framework

✓

Proceed if: 3+ "Yes" answers + compatible tech stack + clear technical ownership

⚠

Explore further if: 2 "Yes" answers + defined blockers + willingness to address in parallel

ℹ

Not a fit if: 1 or fewer "Yes" answers + incompatible framework or vendor lock-in

Next Step: For qualified leads, proceed to technical discovery call and request system architecture overview.

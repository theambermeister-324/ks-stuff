Documentation

No core alignment patterns between design and dev. Siloed pods build net-new components outside Heart.

No variant usage docs. Governance models exist but aren't surfaced. Processes don't always get utilized.

One North's Phased Approach

Where One North Says You Need to Go

A three-phase roadmap to evolve Heart from UI Kit to Integrated Design System. The destination includes MCP integration and AI-assisted generation.

Phase 1 — Stabilize the Core

Define & Prove Standards

  • Define "system-ready" acceptance criteria
  • Token naming alignment workshop
  • Prioritize top 10–15 most-used components
  • Establish Stable vs Experimental lifecycle
  • Token contract alignment: design ↔ dev
Phase 2 — Strengthen & Standardize

Operationalize Governance

  • Apply criteria to next tier of components
  • Formalize contribution models
  • Reorganize Figma libraries by maturity
  • Lifecycle guardrails & visibility mechanisms
  • Component-level structured changelogs
Phase 3 — Automate & Innovate

AI & MCP Integration

  • Explore MCP integration points
  • Pilot AI-assisted component generation
  • Automated changelog generation
  • CI-based token drift detection
  • Component usage & parity health metrics

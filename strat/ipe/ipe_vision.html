Assessment Framework

From Assessment to Journey: A Step-by-Step Process

Follow these five steps to correctly identify which journey an account belongs to and where they should enter.

1

Score the Account

Rate on all three dimensions: Product System Maturity (1-5), AI & Automation Capability (Low/Med/High), and Interest State (Low/Latent/High)

2

Plot the Quadrant

Use scores to identify the account's current quadrant (A-H). Context depth (Maturity + AI capability) forms one axis; Interest State forms the other.

3

Check Account State Flags

Confirm all five CRM flags: Implemented (Y/N), Trust Status (G/Y/R), Activation Depth, MCP/IPE Interest (Y/N), Strategic Fit (Y/N)

4

Apply the Routing Rule

If Strategic Fit = YES and Implemented = NO ‚Üí Journey 3 (no exceptions). This rule overrides quadrant-based assignment.

5

Confirm Journey & Entry Phase

Match the journey to the appropriate entry phase based on current maturity level and activation depth.

Routing Rule (Non-negotiable)

If Strategic Fit = TRUE and Implemented = NO, the account is Journey 3 by default until activation is complete.

This prevents the most common sequencing mistake: selling platform expansion before activation is real.

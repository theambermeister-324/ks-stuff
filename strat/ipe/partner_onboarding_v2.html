Delivery Model

Assessment-Driven Phase Planning: As covered in Overview, the Maturity and Readiness Assessment determines which phases are needed and in what order. This ensures partners are engaged at the right time with the right skills for each phase.

Partnership Phases

Engagements are phase-gated with clear exit criteria. Each phase aligns to a core platform capability so partners know exactly what accelerates activation of the Knapsack core offering:

Phase 0: Deployment & Security D3/D4 Only

Core capability focus: Governance & Standards (security, access, compliance)

Duration: 8-16 weeks | Engagement: Substantial enterprise deployment

Workstreams: Environment provisioning, Security architecture, SSO/SCIM implementation, Vendor security questionnaire, Penetration testing coordination

Exit Criteria: Environment provisioned & accessible, Security review approved, Network connectivity validated, Compliance attestation complete, InfoSec sign-off obtained

Partner Opportunity:

Phase 0 is typically delivered by Deployment Specialist partners who handle enterprise infrastructure, freeing Knapsack's core team to focus on product capabilities. See Partner Roles tab for Deployment Specialist details.

Phase 1: System of Record Connect

Core capability focus: Connect System of Record

Duration: 16-20 weeks | Engagement: Foundation build

Workstreams: Workspace deployment, Entitlement & access setup, Design-to-code connection, System of record structure

Exit Criteria: Teams can publish without intervention, Access no longer blocking work, System of record established

Phase 2: Governance & Enablement Govern + Enable

Core capability focus: Governance & Standards and Implementation & Enablement

Duration: 12-16 weeks | Engagement: Control Plane activation

Workstreams: Governance model activation, Cross-system rules, Enablement workflows, Change management plan

Exit Criteria: Governance model active, Enablement workflows running, Decisions flowing through system

Phase 3: Adoption & Quality Measure

Core capability focus: Adoption & Quality

Duration: 8-12 weeks | Engagement: Adoption and quality acceleration

Workstreams: Adoption rollout, Quality instrumentation, Change validation framework, ROI measurement

Exit Criteria: Adoption metrics trending up, Quality gates in place, Executive value story documented

After core activation: With the core platform fully activated, the Knapsack Product team leads AI capability enablement initiatives, building on the foundation partners deliver.

Key Insights

  • • Multi-role delivery: Each capability requires coordination across roles
  • • FDE focus: Technical implementation and delivery workflows
  • • CX Partner focus: Governance design and business value
  • • PM coordination: Ensures all capabilities delivered successfully
  • • Deployment Specialist: Enterprise security and compliance
  • • Native App Integration Specialist: SDK wrapping and native component sync

How Roles Work Together

Example: Implementation & Enablement

  • • FDE: Implements workflow enablement (Primary)
  • • CX Partner: Designs adoption strategy (Supporting)
  • • PM: Plans rollout and tracks adoption (Supporting)
  • • Deployment Specialist: Ensures secure deployment (Supporting)

Example: Governance & Standards

  • • CX Partner: Designs governance framework (Primary)
  • • FDE: Implements technical controls (Primary)
  • • Deployment Specialist: Ensures security/compliance (Primary)
  • • PM: Coordinates implementation (Supporting)

This coordination is why agencies (with multiple roles) can deliver complete engagements, while independent contractors typically focus on specific capabilities.

Critical Delivery Principles

  • • Phase-gated delivery: Never sell all phases upfront. Earn the right to the next phase.
  • • Value-based pricing: Never quote hourly rates. Price by outcome, not hours.
  • • Clear exit criteria: Each phase must meet defined success criteria before proceeding.
  • • Trust repair focus: Stabilize first, govern second, expand third.

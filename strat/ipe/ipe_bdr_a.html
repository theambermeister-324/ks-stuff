IPE GTM Vision: Key Points for BDRs

The Core Problem

Most customers have design systems but aren't getting value. "Implemented ≠ Activated" — they've built it but teams aren't using it effectively.

Our Approach

We help customers activate their existing design systems through governance, adoption, and AI integration. We're not replacing their system—we're making it work.

The Opportunity

J3 (Activation) = 60% of ARR — This is where most value lives. Customers with 30-70% adoption who need help getting to full value.

AI/MCP Strategy

AI coding tools are creating new urgency. When AI generates code that doesn't match design system standards, it creates governance gaps. This is a key J4 (Intelligence) signal.

Investment Paths Overview

Use MEDDIC signals to determine the right path. J3 is the primary target (60% of ARR).

Deployment Tiers (Ask Early!)

Deployment tier determines pricing premium. Ask compliance questions early—D3/D4 adds $175-550K.

Quick Path Decision Framework

No sponsor OR no budget? J1 Foundation
High readiness + urgent trigger? J2 Acceleration
Sponsor + budget + blocking pain? J3 ← PRIMARY TARGET (60% ARR)
Mature + AI interest + waiting for trigger? J4 Intelligence

Critical Reminders for BDRs

  • • MEDDIC first — Complete all elements before handover. Investment Path questions supplement—not replace—MEDDIC.
  • • J3 is 60% of ARR — This is the primary target path. Look for "implemented but not activated" signals.
  • • Ask deployment tier early — D3/D4 adds significant premium (+$175-550K). Ask compliance questions in Pre-Qualification.
  • • AI roadmap = J4 signal — If AI/MCP is in their roadmap or a must-have requirement, this signals Intelligence path.
  • • Build the deal room — Required before handover. Include Gong recording link and all MEDDIC data.
